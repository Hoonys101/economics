# 로그 파일 검색 도구 설계 (Log File Search Tool Design)

## 1. 문제점: `run_shell_command`와 파일 경로의 공백

현재 `run_shell_command`를 사용하여 Python 스크립트나 기타 명령어를 실행할 때, 파일 경로에 공백이 포함되어 있으면 명령어가 올바르게 해석되지 않는 문제가 발생하고 있습니다. 이는 특히 Windows 환경에서 두드러지며, `findstr`과 같은 기본 쉘 명령어에서도 유사한 문제가 나타납니다.

예시:
`python C:\Users\Gram Pro\coding\economics\search_log.py`
위 명령어에서 `Gram Pro` 부분의 공백 때문에 `python`이 `C:\Users\Gram`을 파일 경로로 인식하고, `Pro\coding\economics\search_log.py`를 별도의 인자로 처리하여 "파일을 찾을 수 없습니다" 오류가 발생합니다.

이 문제를 해결하기 위해 파일 경로를 큰따옴표로 묶는 시도를 했으나, `run_shell_command` 내부에서 쉘이 이를 다시 해석하는 과정에서 여전히 문제가 발생하고 있습니다.

## 2. 해결 방안: 전용 로그 검색 도구 개발

쉘 명령어의 경로 처리 문제로부터 독립적이고, 더 안정적인 로그 파일 검색 기능을 제공하기 위해 새로운 내부 도구 `search_log_file`을 개발할 것을 제안합니다.

### 2.1. `search_log_file` 도구 API 정의

새로운 도구 `search_log_file`은 다음과 같은 기능을 제공합니다:

*   **기능**: 지정된 로그 파일에서 하나 이상의 키워드를 검색하고, 해당 키워드가 포함된 모든 줄을 반환합니다.
*   **인자**:
    *   `file_path` (필수, 문자열): 검색할 로그 파일의 절대 경로.
    *   `keywords` (필수, 리스트[문자열]): 검색할 키워드 리스트.
*   **반환**: 키워드가 포함된 줄들의 리스트. 각 줄은 원본 파일의 내용을 그대로 유지합니다.

### 2.2. `search_log_file` 도구의 장점

*   **안정성**: 쉘 명령어의 경로 처리 문제로부터 자유롭습니다.
*   **편의성**: 복잡한 쉘 명령어 구문 없이 간단한 함수 호출로 로그 검색을 수행할 수 있습니다.
*   **확장성**: 향후 로그 분석에 필요한 추가 기능(예: 정규식 검색, 특정 시간 범위 검색)을 쉽게 추가할 수 있습니다.

이 설계안에 대해 사용자님의 승인을 기다리겠습니다.
