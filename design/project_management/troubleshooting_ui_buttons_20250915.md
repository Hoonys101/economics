# 트러블슈팅: UI 버튼 동작 문제 (2025년 9월 15일)

## 1. 문제 인식 (Problem Recognition)

웹 UI는 정상적으로 표시되지만, 시뮬레이션 제어 버튼(시작, 일시정지, 재설정, 중지)과 설정 저장 버튼이 동작하지 않습니다. 입력 필드는 활성화되어 있으나, 버튼 클릭 시 아무런 반응이 없습니다.

## 2. 확인 방법 (Verification Method)

문제의 원인을 파악하기 위해 다음 방법들을 통해 정보를 수집합니다.

1.  **브라우저 개발자 도구 - Console 탭 확인**: JavaScript 오류 메시지(빨간색 텍스트)가 있는지 확인합니다. 이는 프론트엔드 스크립트의 문제일 가능성이 높습니다.
2.  **브라우저 개발자 도구 - Network 탭 확인**: 동작하지 않는 버튼을 클릭했을 때, 네트워크 요청(API 호출)이 발생하는지, 발생한다면 어떤 상태 코드(Status Code, 예: 200 OK, 404 Not Found, 500 Internal Server Error)와 응답(Response)을 반환하는지 확인합니다. 이는 백엔드 API 연동 문제일 가능성이 높습니다.
3.  **Flask 서버 로그 확인**: `app.py`가 실행되는 터미널에서 오류 메시지나 경고 메시지가 출력되는지 확인합니다.

## 3. 해결 방법 (Solution Method)

(이 섹션은 문제 진단 후 해결 과정에 따라 업데이트될 예정입니다.)

**잠재적 원인:**
*   **프론트엔드 JavaScript 오류**: 버튼 클릭 이벤트 리스너가 제대로 등록되지 않았거나, 이벤트 핸들러 내부에 오류가 있을 수 있습니다.
*   **API 엔드포인트 불일치**: JavaScript에서 호출하는 API 경로가 `app.py`에 정의된 경로와 다르거나, HTTP 메서드(GET/POST)가 일치하지 않을 수 있습니다.
*   **백엔드 오류**: API 엔드포인트는 호출되지만, 백엔드 처리 과정에서 오류가 발생하여 정상적인 응답을 반환하지 못할 수 있습니다.
*   **`simulation_lock` 문제**: `app.py`의 `simulation_lock`이 너무 오랫동안 잠겨 있거나, 데드락이 발생하여 API 요청 처리가 지연될 수 있습니다.
