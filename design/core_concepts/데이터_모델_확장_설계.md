# 데이터 모델 확장 설계 (Data Model Extension Design)

## 1. `goods.json`에 `storability` 속성 추가

### 1.1. 문제점
- 현재 `goods.json`에는 재화의 기본적인 정보만 포함되어 있으며, 재화의 특성(예: 저장성)을 반영하는 속성이 부족합니다. 이는 가계의 재화 구매 의사결정(예: 비축 구매)을 정교하게 모델링하는 데 한계가 있습니다.

### 1.2. 개선 방안
- `data/goods.json` 파일의 각 재화(good) 객체에 `storability` 속성을 추가합니다.
- `storability`는 0.0 (저장 불가능)부터 1.0 (매우 저장 가능) 사이의 값을 가지는 실수(float)로 정의합니다.
- **예시:**
  ```json
  [
    {
      "id": "food",
      "name": "식량",
      "base_utility": {"survival_need": 10},
      "is_luxury": false,
      "storability": 0.8 // 식량은 저장성이 높음
    },
    {
      "id": "luxury_food",
      "name": "고급 식량",
      "base_utility": {"recognition_need": 15},
      "is_luxury": true,
      "storability": 0.3 // 고급 식량은 저장성이 낮음
    }
  ]
  ```

### 1.3. `HouseholdDecisionEngine` 로직 수정
- `HouseholdDecisionEngine`에서 가계가 재화를 구매할 때 `storability` 속성을 활용하여 비축 구매(stockpiling) 의사결정에 영향을 미치도록 로직을 수정합니다.
- `config.py`에 `HOUSEHOLD_STOCKPILING_BONUS_FACTOR`와 같은 상수를 추가하여 저장성이 높은 재화에 대한 비축 구매 보너스를 조절할 수 있도록 합니다.
- **예시 로직 (개념):**
  ```python
  # HouseholdDecisionEngine 내 _calculate_reservation_price 또는 유사 메서드
  def _calculate_reservation_price(self, household, item_id, current_market_price, goods_data):
      # ... 기존 로직 ...

      storability = next((g['storability'] for g in goods_data if g['id'] == item_id), 0.0)
      stockpiling_bonus = storability * config.HOUSEHOLD_STOCKPILING_BONUS_FACTOR * (household.needs.get('stockpiling_need', 0) / 100) # 비축 욕구에 비례

      final_price += stockpiling_bonus
      return final_price
  ```

### 1.4. 예상 효과
- 재화의 현실적인 특성을 시뮬레이션에 반영하여 가계의 행동을 더욱 정교하게 모델링할 수 있습니다.
- 비축 구매와 같은 새로운 경제 현상을 시뮬레이션 내에서 관찰하고 분석할 수 있는 기반을 마련합니다.
