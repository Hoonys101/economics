+# Testing Repair and Protocol Alignment Report
+
+## Architectural Insights
+
+1.  **Protocol Adherence in Mocks**:
+    The `IBank` protocol was recently updated to include `get_total_deposits()`, but test doubles (`MockBank`) in several test files (`tests/unit/systems/test_settlement_system.py`, `tests/finance/test_circular_imports_fix.py`) were not updated. This caused widespread `TypeError`s during instantiation. Strict protocol adherence checks in `SettlementSystem` correctly flagged these invalid mocks.
+
+2.  **Unit Consistency in Tests**:
+    *   **Firm Solvency Logic**: The test `test_solvency_logic.py` incorrectly assumed that `Firm.capital_stock_pennies` needed a `* 100` conversion from `ProductionState.capital_stock`. However, `ProductionState` stores values in pennies (integer) by default. The test was corrected to assert equality without conversion.
+    *   **Asset Management**: Tests for `AssetManagementEngine` were passing investment amounts in pennies (e.g., `100.0` representing 100 pennies) but expecting results consistent with dollars (e.g., `1.0` or `100.0` units of purchase power) given the cost configuration. The engine correctly converts dollar-based config costs to pennies. The tests were updated to use `10000` pennies (100 currency units) to match the expected outcome magnitude.
+
+3.  **Variable Scope Safety**:
+    `simulation/components/engines/production_engine.py` contained a `NameError` in an early-exit path where `capital_depreciation` was referenced but not yet defined. This highlights the importance of initializing return values or ensuring variable scope coverage before all return points.
+
+4.  **Resilient Rollback Logic**:
+    `CommandService.rollback_last_tick` failed in unit tests where the mocked registry did not support `delete_layer` (a specific feature of the concrete `GlobalRegistry`). The logic assumed that if an `origin` was present, `delete_layer` would handle the rollback. The fix ensures that if `delete_layer` is not performed (e.g., capability missing), the system falls back to `delete_entry` to correctly revert parameter creation.
