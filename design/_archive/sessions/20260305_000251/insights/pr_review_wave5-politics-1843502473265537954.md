- **Hardcoded Magic Numbers**:
    - `modules/government/politics_system.py`: `self.election_cycle = 100` is hardcoded in `__init__`. This should be loaded from `GovernmentConfigDTO` via `config_manager`.
    - `simulation/firms.py`: `cost_per_point = 10000` is hardcoded in `formulate_lobbying_effort`. While acceptable for a prototype, this should be a configurable parameter (e.g., in `EconomyParams` or `GovernmentConfig`).
- **Payee Identification**:
    - `simulation/firms.py`: The fallback `payee_id = "GOVERNMENT_TREASURY"` assumes the existence of an agent/account with this ID if `government_state.id` is missing. This is a potential point of failure if the string constant doesn't match the registry.

## 💡 Suggestions
- **Configuration**: Move `election_cycle` and `lobbying_cost_per_point` to `GovernmentConfigDTO` to allow dynamic adjustment via scenarios.
- **Settlement Batching**: In `PoliticsSystem._collect_signals`, lobbying payments are processed sequentially for every firm. For large simulations, consider batching these requests or optimizing the `settlement_system.transfer` call if it involves heavy I/O or logging.
