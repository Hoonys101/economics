# 경제 시뮬레이션 설계서: 가계 주체의 재능과 역량 구조

**버전**: v0.2
**작성일자**: 2025-07-05

---

## 1. 개요

이 문서는 거시경제 시뮬레이션에서 **가계(Household)** 주체의 내부 속성 중,  
- `재능(Talent)`과  
- `분야별 역량(Skill)`  
- `노동 욕구(Labor Need)`
의 구조를 구체화하며,  
**정보의 비대칭성**을 포함한 노동시장 및 투자결정 모델 설계의 기초를 제공한다.

---

## 2. 재능 (Talent)

### 2.1 정의
- **고정 불변 값** (선천적 또는 장기 고정 특성)
- 학습 속도 및 최대 가능 역량치에 영향
- 유관 분야 간 전이 효과를 정의
- 외부 관찰 불가 (정보 비대칭 핵심 요소)

### 2.2 속성

| 속성명 | 설명 |
|--------|------|
| `base_learning_rate` | 학습 속도 계수 (기본값: 1.0, 범위 예시: 0.7~1.3) |
| `max_potential` | 분야별 최대 가능 숙련도 (예: `{"AI": 80, "Finance": 60}`) |
| `related_domains` | 유관 분야 → 전이효과 발생 가능 영역 |
| `visibility` | 외부 노출 불가 (설정 불가, 항상 private) |

### 2.3 예시 구조

```python
class Talent:
    def __init__(self, base_learning_rate, max_potential, related_domains):
        self.base_learning_rate = base_learning_rate
        self.max_potential = max_potential
        self.related_domains = related_domains
```

## 3. 역량 (Skill)

### 3.1 정의
- 특정 분야에 대해 **누적되는 학습 결과**
- 경험, 교육, 자기계발에 의해 증가
- 일부만 외부에 관찰 가능

### 3.2 속성

| 속성명 | 설명 |
|--------|------|
| `domain` | 분야명 (예: "AI", "회계", "기계공학") |
| `value` | 현재 숙련도 수준 (예: 0~100 범위) |
| `observability` | 외부에 보여지는 정도 (0~1) |

### 3.3 예시 구조

```python
class Skill:
    def __init__(self, domain, value, observability=0.5):
        self.domain = domain
        self.value = value
        self.observability = observability
```

## 4. 노동 욕구 (Labor Need) 및 동적 임금 조정

### 4.1 정의
- 가계가 노동 시장에서 일자리를 찾으려는 의지 또는 필요성.
- 일자리를 찾지 못할 경우 증가하며, 임금 협상에 영향을 미친다.

### 4.2 속성

| 속성명 | 설명 |
|--------|------|
| `labor_need` | 현재 노동 욕구 수준 (0~100 범위, 높을수록 일자리가 절실) |

### 4.3 동작 방식

- **`labor_need` 증가**: 매 턴 노동 시장에서 일자리를 얻지 못할 경우 `labor_need`가 일정량 증가한다.
- **`labor_need` 감소**: 일자리를 얻거나, 자산이 충분히 많아질 경우 `labor_need`가 감소한다.
- **희망 임금 조정**: `HouseholdDecisionEngine`에서 가계의 희망 임금(`desired_wage`)을 결정할 때 `labor_need`를 반영한다.
    - `labor_need`가 높을수록 희망 임금을 낮춰서라도 일자리를 얻으려는 경향을 보인다.
    - `desired_wage = (기본 임금) * (1 - labor_need_영향_계수 * labor_need)`

## 5. 가계의 구매 의사결정 및 욕구 확장

가계는 단순히 생존 욕구 충족을 넘어, 시장 가격 변화에 민감하게 반응하여 합리적인 소비를 추구합니다. 이를 위해 '지불 의사 가격' 개념과 '가성비/비축 욕구'를 도입합니다.

### 5.1 지불 의사 가격 (Willingness to Pay) 모델링

각 가계는 자신의 현재 상태(욕구, 자산, 인지된 시장 가격)와 상품의 특성(효용, 저장성)을 바탕으로 각 상품에 대해 최대로 지불할 의사가 있는 가격(`reservation_price`)을 동적으로 계산합니다.

- **계산 요소**:
    - **기본 가치**: 상품의 효용(`utility_per_need`)에 기반한 내재적 가치.
    - **욕구 수준**: 해당 상품이 충족시키는 욕구의 현재 수준. 욕구가 높을수록 지불 의사 가격 증가.
    - **자산 수준**: 가계의 자산이 많을수록 지불 의사 가격 증가.
    - **인지된 시장 가격**: `perceived_avg_prices`를 기준으로 현재 시장 가격이 얼마나 저렴한지 판단.
    - **저장성**: `storability`가 높은 상품은 가격이 저렴할 때 비축하려는 유인이 강해져 지불 의사 가격 증가.

- **의사결정**: 시장 가격이 `reservation_price`보다 낮을 경우 구매를 고려하며, 차이가 클수록 더 많은 수량을 구매하려는 경향을 보입니다.

### 5.2 가성비 욕구 (Price-Performance Desire)

- **정의**: 상품의 절대적 필요성과 별개로, '가격이 싸다'는 사실 자체가 추가적인 소비를 유발하는 욕구.
- **계산**: `(인지된 평균 가격 - 현재 시장 가격) / 인지된 평균 가격` 비율에 비례하여 증가합니다. 이 욕구는 가계의 예산 할당 시 해당 상품에 대한 추가적인 구매력을 부여합니다.
- **AI 통합**: AI의 상태 벡터에 '가성비' 관련 지표를 포함하고, 가성비 좋은 구매 행동에 보상을 부여하여 AI가 이를 학습하도록 유도합니다.

### 5.3 비축 욕구 (Stockpiling Desire)

- **정의**: 상품이 저렴할 때, 당장 필요하지 않더라도 미래 소비를 위해 미리 사두려는 욕구.
- **계산**: `(인지된 평균 가격 - 현재 시장 가격) / 인지된 평균 가격` 비율과 상품의 `storability`에 비례하여 증가합니다. 현재 재고량이 적을수록 이 욕구는 더욱 강해집니다.
- **AI 통합**: AI의 상태 벡터에 '비축 필요성' 관련 지표를 포함하고, 비축 구매 행동에 보상을 부여하여 AI가 이를 학습하도록 유도합니다.

### 5.4 AI 통합 전략

- **상태 (State)**: AI의 상태 벡터에 `perceived_avg_prices`, `storability`, 그리고 계산된 `reservation_price`를 포함하여 AI가 시장 상황과 자신의 구매 의지를 인지하도록 합니다.
- **보상 (Reward)**: `collect_experience` 단계에서 '현명한 소비' 보상(인지된 가격보다 싸게 구매 시)과 '비축 안정' 보상(비축 욕구가 높을 때 구매 시)을 추가하여 AI가 이러한 행동을 학습하도록 유도합니다.

## 6. 가계 주체 (Household Agent) 핵심 속성

가계 주체는 시뮬레이션 내에서 소비와 노동 공급의 주체로서 다양한 속성을 가집니다. 특히, 의사결정 로직을 외부에서 주입받는 `decision_engine` 속성을 통해 유연한 행동 패턴을 구현합니다.

### 5.1 속성

| 속성명 | 설명 | 타입 |
|--------|------|------|
| `id` | 가계의 고유 식별자 | `int` |
| `talent` | 가계의 선천적 재능 (학습 속도, 최대 역량치 등) | `Talent` 객체 |
| `skills` | 가계의 후천적 역량 (학습과 경험을 통해 증가) | `Dict[str, Skill]` |
| `assets` | 가계의 현재 자산 | `float` |
| `inventory` | 가계가 보유한 재화 목록 및 수량 | `Dict[str, float]` |
| `needs` | 가계의 다양한 욕구 수준 (생존, 인정, 성장, 부 축적, 유동성, 노동, 모방, 자녀 양육 등) | `Dict[str, float]` |
| `value_orientation` | 가계의 가치관 (AI 의사결정 모델 선택에 영향) | `str` |
| `current_consumption` | 현재 턴의 총 소비량 | `float` |
| `employer_id` | 현재 고용된 기업의 ID (고용되지 않은 경우 `None`) | `int` 또는 `None` |
| `decision_engine` | 가계의 의사결정을 담당하는 엔진 (AI 또는 규칙 기반) | `HouseholdDecisionEngine` 객체 |
| `goods_data` | 시뮬레이션 내 모든 재화에 대한 정보 | `List[Dict[str, Any]]` |
| `shares_owned` | 보유한 기업 주식 수 | `Dict[int, float]` |
| `is_alive` | 가계의 생존 여부 | `bool` |
| `is_employed` | 가계의 고용 상태 | `bool` |
| `labor_skill` | 가계의 노동 스킬 수준 | `float` |
| `social_status` | 가계의 사회적 지위 | `float` |
| `perceived_avg_prices` | 상품별로 가계가 인지하는 평균 시장 가격 (이동 평균) | `Dict[str, float]` |

## 6. 정보의 비대칭성 구현

### 5.1 외부 관찰 함수

- 실제 숙련도에 확률적 노이즈를 포함
- `observability` 값이 낮을수록 정보 오차가 큼

```python
import random

def observe_skill(skill):
    noise = random.uniform(-1, 1) * (1 - skill.observability)
    return max(0, skill.value + noise)
```

### 5.2 재능의 은폐
- `Talent`는 외부 호출 불가 또는 매우 낮은 신뢰도의 추정치만 가능
- 실질적 판단은 추정치/면접/경험 기반으로 처리 예정
