+# Macro Sensitivity Tuning Insights (Phase 26, WO-062)
+
+## 1. 개요
+본 문서는 `PortfolioManager`에 도입된 거시 경제 연동 위험 회피(Dynamic Risk Aversion) 모델의 민감도 가중치(`INFLATION_STRESS_MULTIPLIER`, `RECESSION_STRESS_MULTIPLIER`)가 시뮬레이션 경제에 미치는 영향에 대한 초기 분석과 향후 튜닝 방향을 제시합니다.
+
+## 2. 민감도 가중치 분석
+
+### 2.1. 현재 설정값
+- `INFLATION_STRESS_MULTIPLIER`: **10.0**
+- `RECESSION_STRESS_MULTIPLIER`: **5.0**
+
+이 설정은 인플레이션율이 목표치(2%)를 1%p 초과할 때마다 위험 회피도를 10%씩 증가시키고, GDP 성장률이 -1%p 기록될 때마다 5%씩 증가시킴을 의미합니다.
+
+### 2.2. 잠재적 영향 분석
+
+#### 긍정적 효과
+- **Flight to Quality 구현**: 현재 설정값은 스태그플레이션과 같은 거시 경제 위기 상황에서 가계가 위험 자산(주식) 비중을 줄이고 안전 자산(예금)으로 이동하는 'Flight to Quality' 현상을 효과적으로 모델링합니다. 이는 `test_portfolio_optimization_under_stagflation` 테스트를 통해 검증되었습니다.
+- **자동 안정화 장치**: 시장 과열(높은 인플레이션) 시 위험 회피 심리가 증가하여 주식 시장의 추가 과열을 억제하고, 경기 침체 시에는 자산 보존 심리를 강화하여 금융 시스템의 안정성을 높이는 긍정적인 피드백 루프를 형성할 수 있습니다.
+
+#### 리스크 및 고려사항
+- **과민 반응 (Over-reaction) 및 시장 붕괴 (Flash Crash)**:
+  - 민감도 가중치가 너무 높을 경우, 일시적인 인플레이션 쇼크나 GDP 하락에도 가계가 과도하게 반응하여 주식을 대량 매도하는 '패닉 셀링(Panic Selling)'을 유발할 수 있습니다.
+  - 모든 에이전트가 동일한 로직으로 동시에 반응할 경우, 주식 시장의 급격한 붕괴(Flash Crash)를 초래할 위험이 있습니다. 이는 시뮬레이션 경제의 비현실적인 변동성을 야기할 수 있습니다.
+- **경기 회복 지연**:
+  - 경기 침체 국면에서 `RECESSION_STRESS_MULTIPLIER`가 너무 높으면, 실제 경제가 회복되기 시작하더라도 가계의 투자 심리가 과도하게 위축되어 실물 경제로의 자금 공급이 지연될 수 있습니다. 이는 경기 회복을 더디게 만드는 요인으로 작용할 수 있습니다.
+- **인플레이션과의 상호작용**:
+  - `INFLATION_STRESS_MULTIPLIER`는 인플레이션 억제 효과를 가질 수 있지만, 동시에 주식 시장 위축으로 인한 기업 투자 감소로 이어져 장기적으로 공급 측면의 인플레이션 압력을 가중시킬 수도 있습니다.
+
+## 3. 튜닝 방향 제언
+
+향후 시뮬레이션 안정성과 현실성을 높이기 위해 다음과 같은 튜닝 방향을 제언합니다.
+
+1.  **에이전트별 민감도 차등화**: 모든 에이전트가 동일한 가중치를 갖는 대신, 가계의 `personality` (e.g., `MISER`, `STATUS_SEEKER`)나 `risk_aversion` 특성에 따라 민감도를 차등 적용하는 방안을 고려할 수 있습니다.
+    - **예시**: `MISER` 성향의 가계는 더 높은 민감도 가중치를, `STATUS_SEEKER`는 더 낮은 가중치를 적용하여 행동의 이질성을 확보합니다.
+
+2.  **비선형적(Non-linear) 가중치 적용**: 현재는 선형적으로 스트레스가 증가하지만, 특정 임계점을 넘었을 때 가중치를 더 급격하게 증가시키는 비선형 함수(e.g., Sigmoid, Exponential)를 도입하여 '패닉' 상태를 더 현실적으로 모델링할 수 있습니다.
+
+3.  **장기 시뮬레이션 통한 검증**: 현재의 단위/통합 테스트를 넘어, 장기(1000+ ticks) 시뮬레이션을 통해 해당 메커니즘이 시장 붕괴나 장기 침체를 유발하지 않는지 스트레스 테스트를 수행해야 합니다. 이를 통해 가중치의 적정 범위를 경험적으로 도출할 필요가 있습니다.
+
+4.  **SoC (Separation of Concerns) 검토**: 현재 `Engine`에서 `MacroFinancialContext`를 집계하여 모든 가계에 동일하게 주입하는 방식은 효율적이지만, 정보 전달의 지연(Lag)이나 비대칭성(Asymmetry)을 모델링하고 있지 않습니다. 향후 고도화 시, 일부 에이전트만 정보를 인지하거나 다른 시간차를 두고 반응하는 모델을 고려하여 SoC를 유지하면서 현실성을 높일 수 있습니다.
+
+## 4. 결론
+현재 구현된 동적 위험 회피 모델과 민감도 가중치는 설계 의도대로 작동함을 확인했습니다. 그러나 장기적인 시장 안정성에 미치는 영향을 고려할 때, 위에서 제안된 방향에 따라 지속적인 모니터링과 미세 조정(Fine-tuning)이 필수적입니다.

+from simulation.dtos import MacroFinancialContext
+from simulation.decisions.portfolio_manager import PortfolioManager
+
+def test_portfolio_optimization_under_stagflation():
+    """
+    Integration test to verify that a household reduces equity allocation
+    under stagflation conditions.
+    """
+    # 1. Setup
+    total_assets = 10000.0
+    risk_aversion = 1.0
+    risk_free_rate = 0.02
+    equity_return_proxy = 0.08
+    survival_cost = 300.0
+    inflation_expectation = 0.02
+
+    # 2. Optimize under normal conditions
+    normal_context = MacroFinancialContext(
+        inflation_rate=0.02,
+        gdp_growth_rate=0.03,
+        market_volatility=0.1,
+        interest_rate_trend=0.0
+    )
+
+    target_cash_normal, target_deposit_normal, target_equity_normal = PortfolioManager.optimize_portfolio(
+        total_liquid_assets=total_assets,
+        risk_aversion=risk_aversion,
+        risk_free_rate=risk_free_rate,
+        equity_return_proxy=equity_return_proxy,
+        survival_cost=survival_cost,
+        inflation_expectation=inflation_expectation,
+        macro_context=normal_context
+    )
+
+    # 3. Optimize under stagflation conditions
+    stagflation_context = MacroFinancialContext(
+        inflation_rate=0.10,
+        gdp_growth_rate=-0.02,
+        market_volatility=0.3,
+        interest_rate_trend=0.01
+    )
+
+    target_cash_stag, target_deposit_stag, target_equity_stag = PortfolioManager.optimize_portfolio(
+        total_liquid_assets=total_assets,
+        risk_aversion=risk_aversion,
+        risk_free_rate=risk_free_rate,
+        equity_return_proxy=equity_return_proxy,
+        survival_cost=survival_cost,
+        inflation_expectation=inflation_expectation,
+        macro_context=stagflation_context
+    )
+
+    # 4. Assert that equity allocation is lower under stagflation
+    assert target_equity_stag < target_equity_normal
+
+    # Also assert that safety preference (cash + deposit) is higher
+    safety_assets_normal = target_cash_normal + target_deposit_normal
+    safety_assets_stag = target_cash_stag + target_deposit_stag
+    assert safety_assets_stag > safety_assets_normal
